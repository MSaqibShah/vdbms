What Do These Metrics Mean?

Create Time: The time it takes to add all documents (including computing their embeddings). Lower is better for fast data ingestion.

Read Time: The time taken to retrieve documents from the store. This is generally fast and similar across models.

Update and Delete Times: The times needed to update or remove a document. These are minor but still important for real-time operations.

Search Time: The time taken to process search queries. Faster search times mean better responsiveness for user queries.

Best Model Recommendation

Based on our aggregated performance metrics, the model sentence-transformers/all-MiniLM-L6-v2 is the best fit for our project.

This conclusion is based on the combined measurement of Create Time and Search Time – two critical factors for our application. For instance, sentence-transformers/all-MiniLM-L6-v2 has an average Create Time of 1,659,641,218 ns and an average Search Time of 45,386,887 ns, which are the lowest among the tested models. Additionally, the low standard deviations indicate consistent performance across runs.
